Why Cilium?

FeatureCiliumFlannelCalico
kube-proxy replacement✅ eBPF replaces iptables entirely❌Partial
Network observability✅ Hubble UI — flow-level visibility❌Limited
NetworkPolicy✅ + FQDN policies, CiliumNetworkPolicy❌✅
L2 announcements (MetalLB replacement)✅ Built-in, no conflict❌❌
Talos compatibility✅ Talos disables kube-proxy — Cilium is the intended CNI⚠️⚠️
Performance✅ XDP + TC eBPF, ~2-5× lower latency than iptablesBaselineBetter than flannel

Note: Cilium's L2 announcements (CiliumL2AnnouncementPolicy) replace MetalLB's ARP layer entirely. MetalLB still allocates IPs; Cilium handles the ARP — no conflict with its own eBPF programs.

SecureBoot — Why it shows ✗ False

Talos SecureBoot requires the installer-secureboot image variant. We used the standard installer, so Talos never enrolled its own UEFI keys.

⚠ To enable: put the M2's BIOS into Setup Mode (clear SecureBoot keys), then run the upgrade command below. Talos will enroll its own keys on first boot.
# Enable SecureBoot (requires BIOS in Setup Mode first)
talosctl upgrade --nodes 192.168.1.10 \
  --image factory.talos.dev/installer-secureboot/36cd6536eaec8ba802be2d38974108359069cedba8857302f69792b26b87c010:v1.13.2

For a homelab with physical access control, this is optional. Talos is still immutable and API-only even without SecureBoot.

Key Commands

Cluster health

kubectl get nodes
kubectl get pods -A
talosctl health --nodes 192.168.1.10

Approve kubelet CSRs (run after node reboots)

kubectl get csr | grep Pending | awk '{print $1}' | xargs kubectl certificate approve

vcluster access

vcluster connect dev --namespace vc-dev
vcluster connect staging --namespace vc-staging
vcluster connect prod --namespace vc-prod
vcluster disconnect   # return to host cluster

Talos — regenerate & apply config

talosctl gen config homelab https://192.168.1.10:6443 \
  --with-secrets talos/secrets.yaml \
  --config-patch-control-plane @talos/patches/controlplane.yaml \
  --config-patch @talos/patches/all-nodes.yaml \
  --config-patch @talos/patches/m2-node.yaml \
  --output talos/ --force
sed -i '' 's/^auto: stable$/hostname: m2/' talos/controlplane.yaml
talosctl apply-config --nodes 192.168.1.10 --file talos/controlplane.yaml

Remaining Setup

1. 1Password Connect (run once)

./scripts/bootstrap-1password.sh
# Requires 1Password desktop app open and authenticated
# Creates Connect server, generates token, creates K8s secrets

2. Tailscale Operator

# Get OAuth creds from: https://login.tailscale.com/admin/settings/oauth
# Scopes needed: devices:core:write, dns:read, routes:read, routes:write
helm repo add tailscale https://pkgs.tailscale.com/helmcharts
helm install tailscale-operator tailscale/tailscale-operator \
  --namespace tailscale --create-namespace \
  --values infrastructure/tailscale-operator/values.yaml \
  --set oauth.clientId=YOUR_CLIENT_ID \
  --set oauth.clientSecret=YOUR_CLIENT_SECRET

# Tailscale punches through AT&T NAT via DERP relay (port 443 outbound).
# No port forwarding required.

3. GPU Node (boot from USB)

# Download ISO (includes NVIDIA extensions):
# https://factory.talos.dev/image/014aaff51f7e1b02c385aad275bf2f4496b0a6e78469710db7f2b8d2a4f8d9dd/v1.13.2/metal-amd64.iso
#
# Flash to USB, boot GPU node, then:
talosctl apply-config --insecure --nodes 192.168.1.101 --file talos/worker-gpu.yaml
# Wait ~3 min, then node joins cluster automatically

4. SimpleFin Exporter (needs custom image rebuild)

# Source was at: ~/repos/simplefin-bridge-exporter-modified/ (old server)
# Upstream: https://github.com/eduser25/simplefin-bridge-exporter
# After rebuild, update image in apps/simplefin/simplefin-cronjob.yaml
# Add real setup token to apps/simplefin/simplefin-secret.yaml
kubectl apply -f apps/simplefin/simplefin-secret.yaml
kubectl apply -f apps/simplefin/simplefin-cronjob.yaml
